Flask Tutorial

Unittest

Something that is untested is broken.

为什么需要进行单元测试呢？代码不是写完了，功能已经完成了么？代码不能止步于功能完成，要让代码更好，可读性，健壮性，都要更好，那单元测试就是让我们代码的健壮性大大提高。大量的开源项目都是包含单元测试的，比如django的测试代码就有几十万行，那如此庞大的测试，有什么好处呢？

  1. 提高项目的健壮性
    通过测试，我们的项目不再那么脆弱，跑通测试，意味着当前代码的健壮性比较高。更高的测试覆盖率，也意味着项目的可靠程度比较高，是值得信赖的。
  2. 提高开发效率
    当修改了代码之后，可以通过跑测试，及时发现问题并得到修复，所以这极大的方便了后续的代码的开发与维护。

初始化

在项目中，增加一个专门进行单元测试的包tests，所有的测试代码都放在这里。
Flask对测试有良好的支持，它暴露了一个Werkzeug的测试客户端，这里，我们直接使用python标准库中的unittest包进行测试。
创建一个测试类： 

class TodoTestCase(unittest.TestCase):  

    def setUp(self):  
        self.app = app.test_client()  

    def tearDown(self):  
        pass  

    def test_index(self):  
        rv = self.app.get('/')  
        assert "awesome-flask-todo" in rv.data

创建了第一个测试方法，当返回的结果中包含“awesome-flask-todo”字样，测试方法就是成功的，当然你可以选择别的特征字样。setUp和tearDown方法，会分别在测试方法运行的开始和结束时运行，在setUp方法中，创建了测试客户端，那在每个测试方法中，我们可以使用这个客户端进行测试。 在项目的目录下，直接运行一下命令，就可以跑起测试用例,discover会自动找到测试包下的测试类并运行其中的测试用例。 

python -m unittest discover

更多的测试用例

需要写更多的测试用例，覆盖view中的所有方法。 

import unittest
from app import app
from app.models import Todo
from werkzeug.exceptions import HTTPException


class TodoTestCase(unittest.TestCase):

    def setUp(self):
        self.app = app.test_client()

    def tearDown(self):
        # clear data for each test
        todos = Todo.objects.all()
        for todo in todos:
            todo.delete()

    def test_index(self):
        rv = self.app.get('/')
        assert "awesome-flask-todo" in rv.data

    def test_empty(self):
        rv = self.app.get('/')
        assert "No todos, please add" in rv.data

    def test_add_todo(self):
        self.app.post("/add", data=dict(content="test add todo"))
        todo = Todo.objects.get_or_404(content="test add todo")
        assert todo is not None

    def test_none_todo(self):
        try:
            todo = Todo.objects.get_or_404(content='test todo none')
        except HTTPException as e:
            assert e.code == 404

    def test_done_todo(self):
        todo = Todo(content='test done todo')
        todo.save()
        url = '/done/'+str(todo.id)
        rv = self.app.get(url)
        assert '/undone/'+str(todo.id) in rv.data

    def test_undone_todo(self):
        todo = Todo(content='test undone todo')
        todo.save()
        url = '/undone/'+str(todo.id)
        rv = self.app.get(url)
        assert '/done/'+str(todo.id) in rv.data

    def test_delete_todo(self):
        todo = Todo(content='test delete done')
        todo.save()
        url = '/delete/'+str(todo.id)
        rv = self.app.get(url)
        assert "No todos, please add" in rv.data

    def test_404(self):
        rv = self.app.get('/404test')
        assert "Not Found" in rv.data

需要注意几个问题：

  1. 测试用例的运行顺序是不确定的。所以，不要让方法之间有依赖。
  2. tearDown清除数据，防止测试方法之间数据有影响。

覆盖率

使用coverage.py来统计测试代码覆盖率，可以更加直观的体现测试的完备度。在项目目录下运行如下命令，跑测试 

coverage run -m unittest discover

查看覆盖率结果 

coverage report
