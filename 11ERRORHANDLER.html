ErrorHandler

在我们的项目中，如果方便访问一个我们在views.py中没有的URL，就会爆出如下错误：
这样是很不友好的，用户看到这种界面，体验非常差，那我们需要给这些错误异常增加处理，以更加友好的方式展示错误。 flask中提供了errorhandler，方便我们处理应用中出现的异常，对于404 not found这种HTTP错误，我们可以在views.py中这样处理: 

@app.errorhandler(404)  
def not_found(e):  
    return render_template('404.html'), 404

然后，我们自定义一个更加用好的界面，404.html： 

{% extends "base.html" %}  
{% block content %}  
    <h2 class="label-warning">Not Found</h2>  
{% endblock %}

然后，我们运行项目，再次访问刚才的url
这样，给人的感觉是不是好很多？当然你可以定义更多的错误异常处理，让应用的体验更好。
